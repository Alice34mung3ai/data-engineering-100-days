
🚀  My 100-Day Data Engineering Journey

🌍 Background & Motivation

My name is Alice Mungai, and I am pursuing a Bachelor’s degree in Applied Computing at KCA University. My path into data engineering began during a student engagement program in our campus ICT office. I witnessed firsthand the real-world consequences of fragile data systems: missing student marks, unaccounted academic records, and a total lack of data lineage to trace whether failures occurred during entry, storage, or transfer.

This experience sparked a deep curiosity: How do we design resilient data architectures where information is guaranteed to be tracked, validated, and safely moved? I didn't have the terminology then, but I now know this is the core mission of Data Engineering. I have committed to a structured 100-day roadmap to master these exact mechanics.

🧭 Phased Focus Areas

My curriculum progresses across four distinct phases, culminating in a production-ready, end-to-end local data pipeline (Ingest → Store → Transform → Orchestrate → Test → Document → Dashboard):

•	🗄️ Phase 1 (Days 1–25) — Foundations, SQL & Data Modeling: OLTP vs. OLAP paradigms, ETL vs. ELT frameworks, advanced SQL (Window functions, CTEs, complex Joins), data normalization, and Kimball dimensional modeling (Star/Snowflake schemas) to directly address the record-keeping issues I witnessed.

•	🐍 Phase 2 (Days 26–50) — Programming, Automation & Linux: Bash scripting, Git/GitHub version control, production Python fundamentals, Pandas data manipulation, database connectivity (psycopg2/SQLAlchemy), and isolated containerization via Docker.

•	⚡ Phase 3 (Days 51–75) — Big Data & Distributed Computing: Distributed computing theory (Hadoop/NoSQL architecture), large-scale data transformations using PySpark, optimized columnar storage formats (Parquet, ORC, Avro), and programmatic pipeline orchestration using Apache Airflow.

•	☁️ Phase 4 (Days 76–100) — Cloud Data Engineering & Capstone: AWS core cloud infrastructure (S3 object storage, EC2 compute instances, boto3 SDK), Cloud Data Warehousing, analytical engineering and data testing using dbt, and final deployment of an end-to-end pipeline.
📚 Learning Approach & Methodology

To balance this journey alongside my university coursework, I am adhering to a strict, action-oriented daily routine:

•	⏱️ Daily Allocation: 4 hours per day split intentionally into: Learn (45 mins) ➔ Practice (90 mins) ➔ Build (60 mins) ➔ Review (25 mins).

•	🐳 Local-First Architecture: Developing on a local stack (Docker, PostgreSQL, Python 3.11+, Git) to master core infrastructure constraints before cloud deployment.

•	🏋️ Practical Labs: Solving algorithmic and query challenges via SQLBolt, HackerRank, LeetCode, and Exercism; ingesting live public data via various APIs (CoinGecko, OpenWeather, World Bank).

•	📖 Canonical Texts: Anchoring my practical skills in industry literature, including Designing Data-Intensive Applications (Kleppmann) and The Data Warehouse Toolkit (Kimball).

•	🇰🇪 Localization: Validating technical concepts against infrastructure realities in Africa (connectivity constraints, compute costs), starting with mock university record architectures and scaling toward projects like Ujima SACCO data models.

🎯 Strategic Goals

•	Model and deploy a fully normalized relational PostgreSQL database backend from a self-designed ERD.

•	Author robust Python scripts to extract API data, execute schema enforcement with Pandas, and load data seamlessly into target tables.

•	Containerize the entire execution pipeline using Docker Compose and automate runtime schedules.
Long-Term Milestones (Day 100 Outcome)

•	Graduate the challenge with a portfolio-grade, publicly documented GitHub repository containing clean code, detailed architectural diagrams, and a recorded system demonstration.
